# VIviDaily Design Brief

> 最后更新：2026-04-10
> 状态：已确认 ✅

---

## 1. Feature Summary

VIviDaily是一个AI热点新闻聚合日报网页应用，包含4个核心页面：
- **Landing Page**：入口页，一个按钮触发爬取昨天一天AI热点
- **Feed Page**：主页面，三栏布局展示新闻流、Top5、AI总结
- **收藏夹页面**：三列卡片展示已收藏内容
- **登录抽屉**：左侧弹出，邮箱密码注册/登录

目标用户：AI从业者、研究者、投资者、产品经理、开发者。使用场景：深夜/办公室环境，快速浏览获取信息。

---

## 2. Primary User Action

**核心行为**：用户打开页面后，快速扫一眼Top5热点榜和AI总结，判断今天AI领域发生了什么重要的事，然后浏览新闻流找到感兴趣的内容，收藏或导出到Notion。

---

## 3. Design Direction

| 维度 | 方向 |
|------|------|
| **视觉基调** | Dark Tech Dashboard |
| **信息密度** | 高密度优先，一屏展示尽可能多内容 |
| **交互风格** | 功能优先，每个元素都有明确作用 |
| **情感目标** | "给懂技术的人用的工具" |

**反参考**：
- ❌ 过度设计、花里胡哨装饰
- ❌ AI生成感、模板化
- ❌ 杂志风格

---

## 4. Layout Strategy

| 页面 | 布局 |
|------|------|
| **Landing Page** | 极简居中，单按钮触发 |
| **Feed Page** | 三栏：左菜单(15%) + 中新闻流(60%) + 右辅助(25%) |
| **收藏夹页面** | 两栏：左菜单(15%) + 右三列卡片(85%) |
| **登录抽屉** | 左侧滑出，背景模糊 |

**视觉层级**：
1. Top5热点榜（第一眼聚焦）
2. AI总结（快速了解）
3. 新闻流卡片（主要内容）
4. 左侧菜单（导航辅助）

---

## 5. Key States

| 状态 | 说明 |
|------|------|
| Landing-默认 | Logo + 标语 + 按钮 |
| Landing-爬取中 | 按钮 Loading |
| Landing-成功 | 自动跳转 Feed Page |
| Landing-失败 | Toast 提示，可重试 |
| Feed-默认 | 新闻流 + Top5 + AI总结 |
| Feed-筛选中 | 新闻流按分类刷新 |
| Feed-搜索 | 提交后显示搜索结果 |
| Feed-已登录 | 菜单显示用户头像 |
| 收藏夹-有收藏 | 三列卡片展示 |
| 收藏夹-无收藏 | 空状态提示 |
| 登录-未登录触发 | 收藏/导出时自动弹出 |
| 登录-登录中 | Loading 状态 |
| 登录-失败 | Toast 提示错误 |

---

## 6. Interaction Model

**卡片交互**：不点击展开，直接点击按钮操作

| 操作 | 交互 |
|------|------|
| 查看原文 | 新标签页打开原文链接 |
| 分享 | 复制链接 → Toast "已复制" |
| 导入Notion | 未登录弹出登录 → 登录后写入 → Toast "已导入" |
| 收藏⭐ | 星标变黄 → 加入收藏夹 → 未登录弹出登录 |
| 取消收藏⭐ | 星标变灰 → 从收藏夹移除 |
| 搜索 | Enter提交 → 新闻流显示结果 |
| 菜单切换 | 新闻流按分类刷新，当前项高亮 |
| 进入收藏夹 | 左侧菜单点击切换页面 |

---

## 7. Content Requirements

| 内容 | 说明 |
|------|------|
| 卡片标题 | 大字体粗体，最多2行截断 |
| 卡片标签 | 来源 + 技术 + 类型，彩色小徽章 |
| 卡片时间 | 如"3小时前" |
| 卡片摘要 | AI概述，最多2-3行截断 |
| Top5 | 5条最热新闻，标题 + 热度分数 |
| AI总结 | 3点热点概述 |
| 菜单项 | Hot、商业、模型、产品、政策、收藏夹、登录 |
| Toast | "已复制"、"已导入"、"收藏成功"、"登录成功" |
| 空状态 | 收藏夹无收藏提示 |
